## WalkScape Advisor — Mission Control Integration

### What We Built
Flask dashboard at http://localhost:5050 with a **WalkScape Advisor** panel in the Automations section:
- Upload WalkScape JSON export → player data parsed and displayed
- Skills grid (12 skills: agility, mining, foraging, woodcutting, fishing, carpentry, cooking, crafting, smithing, trinketry, hunting, tailoring)
- AI recommendation engine (via `openclaw agent` subprocess) for next best activity
- Chat Q&A inside the advisor card
- Player data + recommendation auto-saved to `walkscape_advisor.json` (persists across restarts)

### Files
- `mission_control.py` — Flask backend (port 5050)

### Lessons Learned
- Flask session state (in-memory dict) clears on server restart → saved JSON files are the source of truth
- `loadSavedRecommendation()` must be called on page init AND must independently load player name + skills from the saved JSON
- Logo broken because external wiki URL was hotlinked → downloaded locally to `/static/ws-logo.png`
- The Jinja2 auto-generates empty placeholder cards for items in the `automations` list → WalkScape UI needed to be conditionally merged into the loop
